# Test Folder

This folder contains all testing-related documents and artifacts for StudyPlanner.
The focus is on manual testing performed by the team members.

## Structure
docs/ – Test plan, test cases, test reports, and bug logs.  
data/ – Sample inputs and test data files used during testing.  
reports/ – Session notes, test execution results, and summary of findings.  

## How to Use
1. Start by reviewing the "test plan" in `docs/test-plan.docx` to understand the scope.  
2. Follow the "test cases" in `docs/test-cases.docx` when running tests.  
3. Record results in `docs/test-reports.docx` and log any bugs in `docs/bug-log.docx`.  
4. Refer to `reports/summary.docx` for the overall outcomes of testing
